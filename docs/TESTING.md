# Тестирование

## Команда

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\validate.ps1
```

Скрипт выполняет три проверки:

1. `compileall rental_manager tests` — синтаксис и импорт модулей.
2. `python -m unittest discover -s tests -v` — unit-тесты.
3. Smoke-test на временной SQLite-базе.

Если нужны только unit-тесты:

```powershell
.\.venv\Scripts\python.exe -m unittest discover -s tests -v
```

## Что покрыто

- перенос дат платежа 29/30/31 на 1 число следующего месяца;
- генерация арендных периодов;
- статусы двухчастной аренды;
- расчёт даты отсрочки по количеству дней;
- расчёт полных месяцев проживания;
- ступенчатый тариф по общему расходу объекта;
- распределение ОДН между занятыми квартирами;
- коммунальный расчёт по средней цене объекта;
- ошибка, когда квартирные счётчики больше общего;
- интерполяция и прогноз показаний;
- парсинг тарифных ступеней из UI-строки.

## Демо-набор

Для ручной проверки интерфейса можно заполнить локальную базу фиктивными данными:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\seed-demo.ps1 -ResetDemo
```

После этого в пульте должны появиться:

- просрочка аренды;
- частичная аренда;
- платежи на сегодня;
- отсрочка;
- просроченная коммуналка;
- прогнозный коммунальный счёт;
- личные расходы к компенсации;
- подозрительный чек;
- устаревшие показания.

Это не заменяет автотесты, но позволяет быстро проверить, что интерфейс не выглядит пустым музеем кнопок.

## Как устроены тесты

Тесты используют временную SQLite-базу в `tempfile`. Реальная база `data/rental_manager.db` не трогается.

Базовый набор объектов создаётся через `seed_if_empty`, затем тесты добавляют жильцов, договоры, показания и проверяют расчёты.

## Когда добавлять тест

Тест нужен каждый раз, когда меняется:

- календарная логика аренды;
- статусы долгов и оплат;
- коммунальный расчёт;
- распределение ОДН;
- парсинг тарифов;
- сценарий заезда или выезда.

То есть почти всегда, когда есть деньги и даты. Деньги и даты редко прощают “и так понятно”.
