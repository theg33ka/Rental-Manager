# Прямая интеграция DeepSeek API

Rental Manager обращается напрямую к официальному OpenAI-compatible API DeepSeek. Промежуточный Hermes gateway, Amvera LLM Inference и fallback-провайдеры не используются.

## Настройка через web-панель

1. Откройте `Настройки` → `AI`.
2. Вставьте API-ключ DeepSeek.
3. Выберите модель:
   - `deepseek-v4-flash` — быстрее и дешевле, рекомендуется по умолчанию;
   - `deepseek-v4-pro` — для более сложного анализа.
4. Включите AI-функции и сохраните настройки.

Ключ шифруется тем же механизмом, что Telegram-токены, и не возвращается в API. Пустое поле при повторном сохранении не меняет уже записанный ключ. Если `RENTAL_MANAGER_SETTINGS_ENCRYPTION_KEY` не задан, приложение автоматически создаёт Fernet-ключ в постоянном хранилище. В контейнере используется `/data/rental-manager-settings.key`.

## Управление функциями и расходами

Во вкладке AI раздельно настраиваются:

- модель обычных диалогов и модель операционного автоаудита;
- включение ответов арендаторам на свободный текст;
- включение автоаудита, его периодичность, день недели и время по Новосибирску;
- месячный бюджет, дневной предел запросов и общий предел токенов одного ответа;
- отдельный предел токенов автоаудита и срок действия предложений на подтверждение;
- расчётный курс USD/RUB для консервативной оценки расходов.

Кнопка `Проверить сохранённый API-ключ` выполняет один минимальный запрос к выбранной модели. Этот вызов учитывается в статистике, дневном лимите и бюджете.

Значение `0` отключает месячный бюджет или дневной предел. При достижении ненулевого лимита новые AI-вызовы блокируются, а штатные команды и шаблонные процессы продолжают работать.

Автоаудит не зависит от автоматических сообщений арендаторам. Он запускается только при включённых AI-функциях и автоаудите, после заданного времени и в разрешённый расписанием день. Сообщение владельцу отправляется только при наличии актуальных операционных задач.

## Адаптация промтов

Панель содержит три независимых поля дополнительных инструкций:

- диалоги владельца;
- диалоги с арендаторами;
- ручной и автоматический операционный аудит.

Пустое поле означает системное поведение по умолчанию. Дополнительные инструкции добавляются к защищённому системному промту, а не заменяют его. Они не могут отключить формат JSON, изоляцию данных арендаторов, обязательное подтверждение изменений и backend-валидацию операций.

## Настройка через environment

Environment имеет приоритет над значениями web-панели:

```env
AI_ENABLED=1
AI_TENANT_FREE_TEXT_ENABLED=1
AI_SUPERVISOR_ENABLED=1
AI_SUPERVISOR_CADENCE=daily
AI_SUPERVISOR_WEEKDAY=0
AI_SUPERVISOR_TIME=10:00
AI_SUPERVISOR_MODEL=deepseek-v4-flash
AI_SUPERVISOR_MAX_TOKENS=1200
AI_DAILY_CALL_LIMIT=100
AI_MAX_OUTPUT_TOKENS=1500
AI_USD_RUB_RATE=100
DEEPSEEK_API_KEY=<ключ DeepSeek>
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_REQUEST_TIMEOUT_SECONDS=60
AI_MONTHLY_BUDGET_RUB=1000
```

Поддерживаемые модели ограничены актуальными публичными моделями DeepSeek V4: `deepseek-v4-flash` и `deepseek-v4-pro`.

## Безопасность и действия

- DeepSeek получает подготовленный backend-контекст, а не прямой доступ к базе.
- Изменения данных выполняются только через существующие owner operations.
- Опасные и внешние действия требуют подтверждения владельца кнопкой.
- API-ключи и заголовки авторизации не записываются в логи.
- Расходы учитываются в `ai_usage_daily` и ограничиваются месячным бюджетом.

## Проверка

1. Сохраните ключ и модель во вкладке AI.
2. Убедитесь, что статус показывает `DeepSeek key сохранён`.
3. В owner-чате задайте вопрос о текущих долгах.
4. Проверьте раздел внутреннего мониторинга: провайдер должен быть `deepseek`, модель — выбранная V4.
